# 08 - Limitations

## Purpose

This document identifies the limitations that affect the development of Finance Platform. These limitations influence the project's scope, implementation decisions and overall timeline, helping to ensure that the Beta release remains realistic and achievable.

---

# Introduction

Every software project is developed within a set of constraints. These may include limitations in time, resources, technical knowledge or project scope.

Finance Platform is being developed as an independent project alongside university studies, part-time employment and internship applications. As a result, the Beta release focuses on delivering a stable and reliable foundation rather than attempting to implement every planned feature.

Recognising these limitations early allows realistic decisions to be made throughout development and helps prevent unnecessary complexity.

---

# Project Limitations

| Limitation | Description | Impact |
|------------|-------------|--------|
| **Time Availability** | Development must be balanced alongside university coursework, part-time employment, internship applications and personal commitments. | Limits the amount of development time available each week and requires careful prioritisation of features. |
| **Solo Development** | The project is designed, implemented, tested and documented by a single developer. | All responsibilities fall to one individual, reducing the speed of development compared to a team project. |
| **Beta Scope** | The Beta release focuses solely on tracking and understanding wealth. Advanced features have been intentionally excluded. | Allows development to concentrate on core functionality before introducing additional complexity. |
| **Manual Data Entry** | Asset values must be updated manually by the user. | Users are responsible for keeping financial information accurate and up to date. |
| **No External Integrations** | The application does not connect to banks, brokerages or financial APIs. | Financial information cannot be updated automatically and must be entered manually. |
| **No Real-Time Data** | Investment prices, property values and other financial information are not updated automatically. | Historical records depend entirely on user input. |
| **Limited User Testing** | Initial testing will primarily be carried out by the developer, followed by a small group of friends. | User feedback may not represent every type of future user. |
| **Technology Experience** | The project is developed using technologies already familiar to the developer. | Some newer technologies and frameworks have intentionally been postponed until later versions. |

---

# Future Considerations

Many of the limitations identified within the Beta release are temporary and can be addressed in future versions of Finance Platform.

Potential future improvements include:

- Automatic financial data through external APIs.
- Live investment and property valuations.
- Automated notifications and reminders.
- AI-assisted financial insights.
- Mobile application support.
- Cloud deployment.
- Additional asset categories and integrations.

These enhancements have been deliberately postponed to ensure that the Beta release remains focused on solving the project's primary objective: providing users with a clear understanding of their long-term wealth.

---

## Summary

### Key Points

- The Beta release has been intentionally limited to maintain a realistic project scope.
- Time, available resources and solo development influence the pace of development.
- Manual data entry and the absence of external integrations reduce technical complexity.
- Many limitations identified within the Beta release provide opportunities for future development.
